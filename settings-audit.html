Accessibility Audit: Thunderbird Settings & Configuration

December 2025 - April 2026

A multi-methodology accessibility audit of Thunderbird's Settings interface, evaluated against WCAG 2.2 AA standards. Using the Colour Contrast Analyser, keyboard-only navigation, and VoiceOver screen reader testing, this audit documents color contrast compliance, interactive component state visibility, focus management, keyboard operability, and screen reader compatibility across all major Settings panels — in both light and dark mode. Findings include remediation guidance for identified failures and serve as a before-and-after reference for Thunderbird's upcoming interface update.


Testing was conducted on macOS Tahoe 26.2 to 26.3.1 using Thunderbird 128.5.2 to 149.0.2, MacBook Air built-in keyboard, and VoiceOver.

Thunderbird Settings: General

Key findings included systemic color contrast failures (WCAG 1.4.3 and 1.4.11) across default tag labels in multiple interaction states and interactive controls in both light and dark mode. Dropdown controls deviated from standard keyboard activation patterns, and a missing confirmation dialog for tag deletion introduced risk of irreversible data loss (WCAG 3.3.4). VoiceOver testing revealed flat heading structure and missing group context for form controls.


Passing criteria included focus visibility, logical tab order, radio button and link operability, VoiceOver announcements for buttons, checkboxes, and dropdowns, and reading order consistency.

Thunderbird Settings: Appearance

Key findings included systemic non-text contrast failures (WCAG 1.4.11) affecting dropdowns and buttons in both light and dark mode. A dropdown activated only via arrow keys rather than the standard Enter or Space (WCAG 2.1.1). VoiceOver testing revealed two structural failures: radio button groups with missing or inconsistent group label announcements (WCAG 4.1.2), and four visual section headings — Window Layout, Message List, Cards View Options, and Sorting and Threading — not programmatically defined as headings (WCAG 1.3.1), forcing linear navigation through the panel.


Passing criteria included text contrast in both modes, focus visibility and appearance, logical tab order, checkbox and radio button operability, focus management, reading order consistency, and VoiceOver announcements for buttons, checkboxes, dropdowns, and state changes.

Thunderbird Settings: Composition

Key findings included systemic non-text contrast failures (WCAG 1.4.11) affecting dropdowns and buttons in both light and dark mode. All dropdown controls failed standard keyboard activation, responding only to arrow keys rather than Enter or Space (WCAG 2.1.1). VoiceOver testing surfaced multiple label failures (WCAG 4.1.2) — including dropdowns repeating selected values as their menu name, an unlabeled list box, and radio button options with description text concatenated into the label. Heading structure was flat, with all section headings assigned as level 1 rather than level 2 (WCAG 1.3.1). Two partial failures were also noted: the Font dropdown announced an inaccurate item count, and error messages lacked an alert role and ran together with the window title.


Passing criteria included text contrast in both modes, focus visibility and appearance, logical tab order, checkbox and radio button operability, link activation, focus management, reading order, and state change announcements.

Thunderbird Settings: Privacy & Security

Key findings included systemic non-text contrast failures (WCAG 1.4.11) affecting dropdowns and buttons in both light and dark mode. The panel's sole dropdown failed standard keyboard activation, responding only to arrow keys rather than Enter or Space (WCAG 2.1.1). A partial failure was noted for two adjacent sections identically labeled "DNS over HTTPS," reducing heading navigation efficiency without rising to a full WCAG failure. VoiceOver testing identified two label failures (WCAG 4.1.2): keyserver checkboxes exposed only raw URLs, and a spam setting checkbox omitted its section heading, leaving its purpose ambiguous. Heading structure was flat, with all section headings assigned as level 1 (WCAG 1.3.1).


Passing criteria included text contrast in both modes, focus visibility and appearance, logical tab order, checkbox and radio button operability, link activation, focus management, reading order, role and state announcements, and VoiceOver announcements for buttons, checkboxes, and dropdowns.

Thunderbird Settings: Chat

Key findings included a text contrast partial failure (WCAG 1.4.3) isolating to Red and Blue username labels in the default Bubbles chat theme — message text passed, but username colors fell below 4.5:1 in both modes. Systemic non-text contrast failures (WCAG 1.4.11) were again present across dropdowns and buttons. All dropdown controls failed standard keyboard activation, responding only to arrow keys (WCAG 2.1.1). VoiceOver testing identified a partial label failure on the "Show a notification:" dropdown, which announced only its current value with no field label (WCAG 4.1.2). Heading structure was flat, with all section headings assigned as level 1 (WCAG 1.3.1). A decorative chat preview region was also exposed to VoiceOver, announcing "Patrick, frame" — a meaningless label that should be hidden via aria-hidden (WCAG 1.3.2).


Passing criteria included focus visibility and appearance, logical tab order, checkbox and radio button operability, focus management, role and state announcements, reading order sequence, and VoiceOver announcements for buttons, checkboxes, and headings.

Thunderbird Settings: Export for Mobile

Key findings included a text contrast partial failure (WCAG 1.4.3) on the primary workflow controls — the inactive "Select All," inactive "Export," and active "Export" button labels fell below 4.5:1 in both modes. Systemic non-text contrast failures (WCAG 1.4.11) were again present, though no dropdown or radio button operability issues applied, as neither control type exists on this panel.


Passing criteria included focus visibility and appearance, logical tab order, checkbox and button operability, focus management, all VoiceOver label and structure criteria, reading order, and state announcements.
