Accessibility Audit: Thunderbird Settings & Configuration

December 2025 - May 2026

A multi-methodology accessibility audit of Thunderbird's Settings interface, evaluated against WCAG 2.2 AA standards. Using the Colour Contrast Analyser, keyboard-only navigation, and VoiceOver screen reader testing, this audit documents color contrast compliance, interactive component state visibility, focus management, keyboard operability, and screen reader compatibility across all major Settings panes — in both light and dark mode. Findings include remediation guidance for identified failures and serve as a before-and-after reference for Thunderbird's upcoming interface update.



Testing conducted on macOS Tahoe 26.2 to 26.3.1 using Thunderbird 128.5.2 to 147.0.2, MacBook Air built-in keyboard, and VoiceOver.


*Screen reader compatability testing in progress — findings to be added in a future iteration of this audit.

Full Audit Findings (Google Sheets)

Thunderbird Settings: General

General Settings: Audit Findings (Google Sheets)

The General settings pane serves as the baseline for this accessibility audit. Color contrast testing across light and dark modes found that body text, headings, and links all meet WCAG 2.2 AA requirements — however, default tag label colors fail the 4.5:1 minimum ratio required for normal-sized text. Interactive component contrast failures were also identified across both modes, affecting dropdown and button boundaries. Focus visibility passed in both modes with contrast ratios well above the 3:1 minimum. Pinch-to-zoom is unsupported even though keyboard command zoom functions correctly, which is noted as a platform interaction gap rather than a WCAG failure.

*Screen reader compatability testing in progress — findings to be added in a future iteration of this audit.

Thunderbird Settings: Appearance

Appearance Settings: Audit Findings (Google Sheets)

The Appearance pane presents the same interactive component contrast failures found throughout the Settings interface — dropdown and button boundaries fall below the 3:1 minimum under WCAG 2.2 AA 1.4.11 in both light and dark modes. Body text, headings, and links passed in both modes, focus visibility was strong, and headings were clear and descriptive. Pinch-to-zoom remains unsupported, consistent with the platform interaction gap noted throughout this audit.

*Screen reader compatability testing in progress — findings to be added in a future iteration of this audit.

Thunderbird Settings: Composition

Composition Settings: Audit Findings (Google Sheets)

The Composition pane reflects the systemic interactive component contrast failures found across Settings, with dropdown and button boundaries falling below the 3:1 minimum under 1.4.11 in both modes. Body text, headings, and links passed contrast requirements, and focus visibility was strong in both modes. Of note: the Composition pane contains a higher density of interactive controls than most other panes, making the component boundary failures here more likely to affect user workflows in practice.

*Screen reader compatability testing in progress — findings to be added in a future iteration of this audit.

Thunderbird Settings: Privacy & Security

Privacy & Security Settings: Audit Findings (Google Sheets)

The Privacy & Security pane follows the systemic pattern found across the Settings interface, with one notable exception. Body text, headings, and links passed color contrast requirements in both light and dark modes, while interactive component boundaries — including dropdowns and buttons — fall below the 3:1 minimum required under WCAG 2.2 AA 1.4.11 in both modes. Focus visibility passed with strong contrast ratios in both modes. A unique finding on this pane is a redundant heading label — the "DNS over HTTPS" section heading appears twice consecutively, which may reduce navigation efficiency for screen reader users relying on heading navigation. This was filed as a partial failure under 2.4.6 Headings and Labels rather than a full WCAG failure, as it does not prevent navigation or understanding. Pinch-to-zoom remains unsupported, though keyboard zoom functions correctly.

*Screen reader compatability testing in progress — findings to be added in a future iteration of this audit.

Thunderbird Settings: Chat

Chat Settings: Audit Findings (Google Sheets)

The Chat pane introduced a unique finding not present in other settings panes. While most UI elements passed color contrast testing, the default Bubbles chat theme applies Red and Blue username display colors that fail the 4.5:1 minimum ratio required for normal-sized text under WCAG 2.2 AA 1.4.3 — in both light and dark modes. Notably, the alternating red and blue message text itself passed contrast testing; the failure is isolated to the username labels specifically. Interactive component boundaries also fail the 3:1 minimum under 1.4.11 in both modes, consistent with the systemic pattern found across all panes. Focus visibility passed with strong contrast ratios, headings were clear and descriptive, and pinch-to-zoom remains unsupported though keyboard zoom functions correctly.

*Screen reader compatability testing in progress — findings to be added in a future iteration of this audit.

Thunderbird Settings: Export for Mobile

Export for Mobile Settings: Audit Findings (Google Sheets)

The Export for Mobile pane introduced a pane-specific finding alongside the systemic failures found throughout the Settings interface. The primary controls for this workflow — the "Select All" and "Export" buttons — fail the 4.5:1 minimum text contrast ratio under WCAG 2.2 AA 1.4.3 in both light and dark modes, affecting both active and inactive button states. As these are the only interactive controls on this pane, the failure creates a direct barrier to completing the Export for Mobile task independently. Interactive component boundaries also fail the 3:1 minimum under 1.4.11 in both modes, consistent with the systemic pattern. Focus visibility passed with strong contrast ratios in both modes, headings were clear and descriptive, and pinch-to-zoom remains unsupported though keyboard zoom functions correctly.

*Screen reader compatability testing in progress — findings to be added in a future iteration of this audit.
